    Represents a UI container.

    This is a top-level layout component that can only be used on LayoutView and can contain ActionRows, TextDisplays, Sections, MediaGallerys, Files, and Separators in it.

    This can be inherited.

    New in version 2.6.

    Examples
    content_copy

    import discord
    from discord import ui

    # you can subclass it and add components as you would add them
    # in a LayoutView
    class MyContainer(ui.Container):
        action_row = ui.ActionRow()

        @action_row.button(label='A button in a container!')
        async def a_button(self, interaction: discord.Interaction, button: discord.ui.Button):
            await interaction.response.send_message('You clicked a button!')

    # or use it directly on LayoutView
    class MyView(ui.LayoutView):
        container = ui.Container(ui.TextDisplay('I am a text display on a container!'))
        # or you can use your subclass:
        # container = MyContainer()

        This function is a coroutine.

        A callback that is called when an interaction happens within this item that checks whether the callback should be processed.

Rules:
Section: has to include at least one Text Display AND an thumbnail or Button Accessory. Cant be just a TextDisplay
Pings: When not edited via {allowed_mentions:{parse:[]}}, it will ping all pings inside. its good practice to parse[], if there is no need to pings someone
General: You cant include Embeds/Content when sending ComponentV2 MessageActionRowComponentBuilder
Sending: When you send componentV2 message: {components: [Array of ContainerBuilders (preferably only one ContainerBuilder per message)], flags: 32768}
Example: to Replicate an Embed you would use (in javascript:) ContainerBuilder().addTextDisplayComponents(new TextDisplayBuilder.setContent("## Title")).addSeperatorComponents(true).addSectionComponents(new SectionBuilder().addTextDisplayComponents(new TextDisplayBuilder.setContent("description").setThumbnailAccessory(new ThumbnailBuilder().setURL("thumnailurl"))...)), You will have to get the gist how to do this in discord.py yourself.
Discord Markdown Guide: "### text" : H3 -> slightly bigger + bolded to normal Text, "## text" : H2 -> significantly bigger than normal text, comparable to embed titles, "# text" : H1 -> Really large text, e.g. for important headlines, "-# text" -> Small text, more greyish for e.g. not so important statements or to keep the menu smaller, "> text" -> grey text + adds a "tab" to texts. this can be combined with size modifications e.g. "## > text" used to e.g. highlight an information, ```css\ntext``` -> syntax highlighting code block, can be easily copypasted by clients, `text` -> inline codeblock, **bold**, *italic*, ~~striketrough~~
